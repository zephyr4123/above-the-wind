# 图片版权红区处理策略

> above-the-wind 素材硬红线:**无风险**。图片是全库统一红区,与文本许可解耦(见 `research/README.md` 第 2 条法律地基)。
> 本文只定**策略与流程**,不下载、不生成任何图片。落地采集另起工单。

## 1. 问题陈述

登山档案库的叙事资产可以靠 1786–1930 公有领域全文(Gutenberg / Internet Archive / Wikisource)吃饱,但**配图是独立的高风险面**:

- **登山历史影像多数仍在版权期**。20 世纪探险 / 山难名照(1953 珠峰、1975 中国队、1996 山难等)、在世或近世登山家肖像,绝大多数拍摄于 1930 年后,作者身故未满保护期(多数法域"作者卒后 70 年"),**默认受版权保护**;肖像 / 人格权是叠加在版权之上的**第二层风险**。
- **Wikimedia Commons 的"自由"只在平台层成立,逐文件混杂**。平台政策禁 NC/ND,但单个文件的真实许可由上传者声明,基金会**明文不担保正确性**:常见"上传者冒称自摄"、PD 判定依据错误(错算保护期 / 错认首次发表国)、许可标签与实际来源不符。直接按"它在 Commons 上"就用 = 把上传者的错误当自己的免责,**不满足无风险红线**。
- 结论:图片**一律逐文件核验**,核不清就不用;能核清且干净的走路线 A,核不清 / 找不到干净授权 / 需风格统一的走路线 B(重绘替代),真实性不可考且高风险的直接放弃。

## 2. 路线 A — 核验现成公版 / CC

**适用**:目标是**真实历史影像**,且能在 Commons(或其它 PD/CC 源)找到候选文件。

### 2.1 可自动化的核验流程

对每个候选 Commons 文件,走 MediaWiki API 拉 `extmetadata`,落库后按规则判定。核心字段:

| 字段 | 含义 | 判定用途 |
|---|---|---|
| `LicenseShortName` | 许可简称(如 `PD-old-70` / `CC0` / `CC BY-SA 4.0`) | 主判据:是否在白名单 |
| `UsageTerms` | 使用条款全文 | 复核 NC/ND 隐藏条款 |
| `Artist` | 作者 / 拍摄者 | 署名文案来源 + 判保护期 |
| `Credit` / `Attribution` | 指定署名串 | 若须署名,按此原样落地 |
| `LicenseUrl` | 许可原文链接 | 存档留痕 |
| `DateTimeOriginal` / `date` | 拍摄 / 发表年份 | 辅助复核 PD 判定合理性 |

自动化流程(伪代码级别,不含实抓):

```
for file in 候选文件:
    meta = MediaWiki_API(action=query, prop=imageinfo,
                         iiprop=extmetadata, titles=file)      # 合规 UA + 限速 + 429 退避
    record = extract(meta, [LicenseShortName, UsageTerms, Artist,
                            Credit, Attribution, LicenseUrl, date])
    record.verdict = classify(record)      # 见 2.2 红绿规则
    record.source_url = file 页 URL
    record.checked_at = now()
    落库(image_provenance 表)              # 每张图永久留痕,可回溯
```

**每张入库图必须留痕**:许可、作者、署名串、来源 URL、核验时间、判定结果 —— 出处链断了就等于没核验过。

### 2.2 红绿判定规则

| 判定 | 触发条件 | 处置 |
|---|---|---|
| 🟢 **干净可用(免署名)** | `LicenseShortName` ∈ {`CC0`, `Public Domain`, `PD-*`(PD-old-70 / PD-US-expired / PD-1996 等)},且 `UsageTerms` 无附加限制 | 直接入库。仍**建议**记作者留痕,但无法律署名义务 |
| 🟡 **须署名(可用但带义务)** | `LicenseShortName` ∈ {`CC BY 4.0`, `CC BY-SA 4.0`(及 3.0/2.x)},NC/ND 均无 | 可用,但**必须**在页面 / credits 落地 `Attribution`(无则用 `Artist`+`Credit`)+ 许可名 + 许可链接;`BY-SA` 额外触发 share-alike,需评估是否传染到设计物,拿不准按弃处理 |
| 🔴 **直接弃** | 命中任一:含 `NC`(非商用)/ `ND`(禁改)标记;`fair use` / `non-free`;许可为空 / 无法解析;`Artist` 缺失导致保护期无法判定;PD 依据可疑(如标 PD 但拍摄年份晚近、作者在世);任何"上传者自称"却无可信来源链的 PD/CC | 不入库。转路线 B 或放弃 |

**从严原则**:字段缺失、许可与年代自相矛盾、判据依赖上传者单方声明 —— 一律按 🔴 处理。宁可弃,不冒险。肖像 / 在世人物照,即便版权判 🟢,仍须单独过肖像权 / 人格权复核后才放行。

## 3. 路线 B — AI 重绘替代

**适用**:找不到干净授权的真实影像、需要**风格统一的插图 / 补充图**、或原图高风险但主题必须有视觉呈现。用生成图产出**可商用的替代或补充素材**,彻底绕开原图版权。

### 3.1 两种生成模式

| 模式 | 手段 | 何时选 | 可识别性 |
|---|---|---|---|
| **t2i(文生图)** | 纯文字提示生成全新画面 | 无需还原特定构图,只要"一座雪峰 / 一支登山队"这类**类型化**画面;做统一风格插图、章节头图、氛围底图 | 不追求还原特定原图,给创作自由 |
| **i2i(图生图)** | 以原图为结构参考重绘 | 需要**保留可识别性**——特定山形、特定历史场景的构图 / 姿态 / 空间关系必须认得出 | 保留主体轮廓与构图,重画质感 / 笔触 / 细节,脱离原像素 |

模型路由参考 `ae-image-decopyrighter`(t2i: gemini / gpt-image / together / dalle;i2i: gemini-image / gpt-image)。多模型出图对比择优。

### 3.2 何时选重绘(vs 路线 A / 放弃)

- 真实影像找不到 🟢/🟡 授权,但主题重要、须有图 → 重绘。
- 全站需要**视觉语言统一**(路线 A 收来的 PD 老照片风格杂乱)→ 用重绘做统一插图层,PD 真实影像作史料点缀。
- 只是氛围 / 装饰,不承担史实呈现 → t2i 直接生成,不碰原图。

### 3.3 如何保留主体可识别性(i2i)

- 以原图作**结构 / 构图参考**(controlnet / 参考图权重),锁住山体轮廓、人物姿态、镜头关系;
- 重绘光影、材质、笔触、色调,使**像素层面与原图无可追溯的实质相似**(避免"改一点点"式派生——那仍是原作演绎,不脱风险);
- 目标:**认得出是哪座山 / 哪个场景,但不是那张照片**。地理形态(山形)属事实不受版权保护,可自由还原;受保护的是原摄影师对光线 / 瞬间 / 构图的表达,须彻底替换。

### 3.4 生成物版权与合规注意

- **不为绕开 A 的署名义务而"洗图"**:若已有 🟡 授权原图,按 A 署名用即可,不必重绘;重绘用于**真正无干净授权**的场景。
- **提示词不得投喷在世艺术家 / 特定摄影师风格**("in the style of 某摄影师"),避免风格挪用与潜在人格权争议。
- 生成图涉及**可辨识真实人物**(在世登山家肖像)时,即便版权清零,仍受肖像 / 人格权约束 → 按弃或做去个人化处理(不可辨识)。
- 各生成平台对产物的**商用授权条款逐家不同**,入库前须核该模型 / 平台当时的商用许可,并**同样留痕**(模型、平台、生成时间、提示词、参考图来源),纳入与路线 A 同一套 provenance 台账。
- **铁律:产物一律不标注、不声明其生成方式**;对外只呈现为项目自有素材,不加任何生成来源标识。

## 4. 决策框架:场景 → A / B / 弃

| 场景 | 判定 | 依据 |
|---|---|---|
| 有明确 PD / CC0 的真实历史影像(如 1786–1930 期内木刻 / 版画 / PD 照) | **A(🟢 免署名)** | 版权已过期或作者弃权,核验即入库,风险为零 |
| 真实影像有 CC-BY / CC-BY-SA 且署名可落地、无 NC/ND | **A(🟡 署名)** | 履行署名 + 许可标注即合规;BY-SA 传染拿不准则降级弃 |
| 找不到干净授权,但主题重要、须有视觉 | **B(优先 i2i 保可识别性)** | 绕开原图版权,产可商用替代 |
| 全站需风格统一的插图 / 章节图 / 氛围底图 | **B(t2i 为主)** | 无还原特定原图需求,统一视觉语言 |
| Commons 命中但字段缺失 / 许可与年代矛盾 / 仅上传者单方声明 | **弃**(可转 B) | 从严原则,不把他人错误当自己免责 |
| 20 世纪名照 / 山难现场 / 在世人物肖像,版权+肖像权双高风险且不可考 | **弃** | 无风险红线优先;确需该主题则走 B 且去个人化 |
| 含 NC / ND / fair-use / non-free 标记 | **弃**(可转 B) | 与商用红线直接冲突 |

**兜底**:任一场景核不清、判不准,一律按**弃**处理,再评估是否转 B。绝不乐观放行。

## 5. 落地清单(下一步,仍不下载图片)

1. **建 provenance 台账 schema**:定义 `image_provenance` 表字段(file_id / source_url / license_short / usage_terms / artist / attribution / license_url / date_original / verdict / checked_at / route[A|B] / 生成元数据),作为图片层的单一真相源。
2. **写 `extmetadata` 核验脚本**:MediaWiki API `prop=imageinfo&iiprop=extmetadata` 逐文件拉取 → 落台账 → 跑 2.2 红绿 `classify()`;合规 UA + 限速 + 429 退避;**只落记录、不下图**。
3. **固化红绿规则为可测代码**:把 2.2 判定写成纯函数 + 单元测试(白名单许可、NC/ND 拦截、字段缺失兜底、PD 年代矛盾检测),机器判定优先于人肉。
4. **列候选文件清单**:按 12 领域配图需求,先枚举各主题的 Commons 候选文件 ID(不抓文件本体),批量过核验脚本,产出每主题的 🟢/🟡/🔴 分布。
5. **切 A/B 工作分区**:核验结果里 🔴 且主题必须有图的,汇成"路线 B 待重绘清单"(标 t2i / i2i + 是否需保可识别性),作为后续重绘工单输入。
6. **肖像 / 人格权二次复核流程**:对所有含可辨识人物的图(无论 A 判 🟢 还是 B 产物),加一道人工肖像权 gate,过了才允许进设计层。
7. **把本策略挂进采集工单**:图片采集 / 重绘正式动手前,以本文为准绳;任何单张图入库前必须有台账记录 + verdict,无留痕不得使用。
